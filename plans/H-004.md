# H-004 plan

## Goal
launch-critical path を早期に確認できる Playwright smoke scaffold と dev-store checklist を repo に固定する。

## Read first
- `tickets/harness/H-004-playwright-and-dev-store-smoke-scaffolding.md`
- `docs/codex_harness_bootstrap.md`
- `tickets/platform/P-001-embedded-shell-and-session-auth.md`
- `tickets/platform/P-003-entitlement-refresh-pricing-gate-and-state-mapping.md`
- `.agents/skills/app-review-readiness/SKILL.md`

## Constraints
- full product parity scenario までは作らない。
- smoke は embedded shell、pricing shell、invalid session path に限定する。
- reviewer path truth を変えない限り ADR は更新しない。

## Steps
1. smoke 対象の最小シナリオを整理し、Playwright の scaffold を追加する。
2. dev-store で再現できる手順を checklist 化してコミットする。
3. `pnpm check` に Playwright の構文健全性確認を組み込み、scaffold が壊れていないことを確認する。

## ADR impact
- ADR required: no
- ADR: none
- Why: reviewer path truth を変えず、smoke scaffold と checklist だけを追加するため。

## Validation
- `pnpm run test:smoke:list`
- `pnpm check`
- checklist の manual review

## Risks / open questions
- 実アプリ本体が未実装のため、Playwright は skeleton として `--list` 検証を基本にする。
- 実ブラウザでの reviewer dry run は P-001/P-003/O-002 の実装後に手順どおり実施する。
