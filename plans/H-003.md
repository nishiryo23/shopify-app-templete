# H-003 plan

## Goal
billing、webhook、CSV provenance の高リスク truth を contract tests で固定する。

## Constraints
- UI や full workflow は作らない。
- billing truth は `currentAppInstallation` 起点の状態マッピングに限定する。
- webhook truth は raw body + HMAC + durable inbox write + duplicate no-op に限定する。
- provenance truth は manifest/file digest と row fingerprint verify に限定する。

## Steps
1. billing / webhook / provenance の truth を小さな contract module に切り出す。
2. `node:test` で acceptance を直接叩く contract tests を追加する。
3. `pnpm check` に contract tests を組み込み、harness gate として実行できるようにする。
4. `pnpm check` を実行して結果を確認する。

## ADR impact
- ADR required: no
- ADR: none
- Why: contract test で既存 truth を固定する計画であり、設計判断自体は変えないため。

## Validation
- `pnpm run test:contracts`
- `pnpm check`

## Risks / open questions
- webhook inbox は in-memory fake で contract を固定し、実 DB 実装は P-004/P-005 で差し替える。
- CSV parsing は最小 contract として改行単位で扱い、列意味論はドメイン ticket で追加する。
