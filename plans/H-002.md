# H-002 plan

## Goal
`pnpm check` と architecture guardrails を整備し、Codex が routes / webhook / Shopify Admin API の境界を壊せない状態を作る。

## Read first
- `tickets/harness/H-002-quality-gates-and-architecture-guardrails.md`
- `docs/shopify_app_technical_spec_complete.md`
- `adr/0001-repo-truth-and-codex-harness.md`
- `adr/0002-embedded-auth-and-token-exchange.md`
- `adr/0004-app-specific-https-webhooks-only.md`
- `.agents/skills/product-domain-parity/SKILL.md`

## Constraints
- feature implementationや business logic は追加しない。
- direct Admin API access、webhook inline business logic、routes から domain 直結を guardrail で禁止する。
- `shopify.app.toml`、scope、billing truth、webhook policy、privacy/delete contract は変更しない。

## Steps
1. guardrail の根拠を docs / ADR から確認し、禁止パターンを明文化する。
2. `package.json` と `pnpm check` を追加し、Node 組み込みだけで実行できる check script を実装する。
3. valid / invalid fixture を追加して、guardrail が通る例と落ちる例を smoke で固定する。
4. ADR-0001 を更新し、harness truth に `pnpm check` と architecture guardrails を追加する。
5. `pnpm check` を実行して成功を確認する。

## Validation
